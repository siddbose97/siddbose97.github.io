A-Maze-ing Solver

Algorithms and GUIs

During my time at UCLA, I always enjoyed working on algorithmic problems but even my more casual projects never had graphical interfaces more extensive than ASCII representations of the real deal (connect 4, tic-tac-toe, etc). As a way of taking a step into the world of GUIs while practicing recursive algorithms (and in the end search algorithms), I decided to undertake a project a bit out of my comfort zone. I sought to create a maze-generating program that would create a maze of a certain size and then offer a solution to the maze. I started out by implementing a recursive approach to a randomized version of Prim's algorithm and then moved onto a DFS approach using stacks/lists in Python to ensure that there would be no overflow issues with bigger mazes.


The most challenging aspect of this project was definitely marrying the graphical interface to the underlying algorithm. Even in my other programming endeavors, I have found myself more comfortable in analytical tasks like Artificial Intelligence algorithm implementations or breaking down and optimizing algorithmic complexities. Ensuring a product is aesthetically pleasing, although a goal of mine, was never the biggest priority for the end product. As such, I spend an inordinate amount of time in this project working on ensuring the display was clean and robust to ensure the experience of watching the maze being generated and solved was pleasing. From the portfolio project in JavaScript to using TKinter in Python for this project I have been enjoying the challenge of working on design and display elements and I hope to keep broadening my horizons in this way.
